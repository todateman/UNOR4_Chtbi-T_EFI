# ECUデバッグ評価レポート

**日付**: 2026-03-30
**対象**: UNOR4_Chtbi-T_EFI (Arduino UNO R4 Minima ベース ECU)
**目的**: INJ_OUT / IGN_OUT タイミング偏差の確認・修正

---

| 機器 | 詳細 |
| ---- | ---- |
| ECU | Arduino UNO R4 Minima (RA4M1) |
| プログラム<BR>(コミットハッシュ) | `#c7301b6b43c53b5172ccfd44263025526725fd54` |
| エンジン信号発生器 | [Ardu-Stim](https://github.com/todateman/Ardu-Stim.git) `develop/furoshiki` ブランチ `furoshiki_2025` パターン |

## 2. 発見した問題

### 2-1. ECUが旧ファームウェアで動作していた

デバッグ初期に取得したシリアルモニタ出力において、`calculatedIGN_CA = 185` が確認された。
現行コード (`main.cpp`) の `defaultMap` における最大値は `145`（4800 RPM以上）であり、**+40°の乖離**が存在した。

| 項目 | 旧ファームウェア | 現行コード |
| ---- | ---------------- | ---------- |
| 最大 IGN_CA | 185° | 145° |
| ISR処理遅れ補正 | MAPに+40°加算 | 不要（ISR最適化済） |

### 2-2. 原因

旧コードでは ISR (`ReadNe_ISR`) 内で浮動小数点演算を行っていたため、実行時間が約40°CA分遅延していた。
この遅れを補正するため、MAP の `ign_ca` に一律+40°が加算されていた。

現行コードでは ISR 内を整数演算のみに最適化し、処理遅れを約1°CA に短縮。
MAP値を補正不要な本来の進角値 (最大145°) に修正済み。

## 3. 対処

### 3-1. ファームウェア書き込み

```sh
~/.platformio/penv/bin/pio run --target upload
```

DFU モードで書き込み成功。

### 3-2. visa-mcp サーバ修正

USB VISA リソース (`USB0::...::INSTR`) に対して `read_termination` / `write_termination` を設定していたため、PyVISA が `'read_termination' is not a valid attribute` エラーを返していた。
`src/server.py` を修正し、USB プロトコル時はターミネーション設定をスキップするよう変更した。

| 評価項目 | 結果 |
| -------- | ---- |
| MAP値との一致（全RPM） | **合格** (偏差0) |
| INJ タイミング (4000 RPM) | **合格** (パルス幅誤差 <1%) |

| INJ タイミング (6000 RPM) | **合格** (パルス幅誤差 <5%) |

## 6. タイミング計算式（参考）

```txt
INJ 開始: Ne_deg >= INJ_STR_CA (= 0°)  ← サイクルリセット直後
IGN 開始: Ne_deg >= (360 - calculatedIGN_CA)
INJ→IGN 期待時間差: (360 - ign_ca) / 360 × (60000 / RPM) ms
```

例 (6000 RPM, ign_ca=145):

```txt
(360 - 145) / 360 × (60000 / 6000) = 215/360 × 10 = 5.972 ms
```

例 (4000 RPM, ign_ca=135):

```txt
(360 - 135) / 360 × (60000 / 4000) = 225/360 × 15 = 9.375 ms
```

> ※ エンジン1サイクル = クランク2回転。サイクル周期 = 2 × (60000/RPM) ms

## 7. 実エンジン適用に向けた総合評価

### 7-1. 本評価の範囲と限界

本評価はArdu-Stimによるエンジン信号エミュレータを使用したものであり、実燃焼環境は含まない。

### 7-2. エミュレータ評価で確認できた項目

| 項目 | 内容 | 評価 |
| ---- | ---- | ---- |
| MAP値の適用 | 全RPMでcommand値と一致 | ✓ |
| IGN タイミング精度 | 偏差 ≦ −3.8°CA（ISR遅れと整合） | ✓ |
| INJ パルス幅精度 | 誤差 <1% | ✓ |
| IGN パルス幅 | 5.0 ms（IGNITION_HOLD_US通り） | ✓ |
| ISR最適化効果 | 旧比 −40°CA の遅れを解消 | ✓ |

### 7-3. エミュレータ評価でカバーできていない項目

#### 制御ロジック面

| 項目 | 懸念事項 |
| ---- | -------- |
| **負荷補正なし** | MAPはRPMのみ。スロットル開度・吸気圧不在のため、実負荷域での空燃比乖離が未知 |
| **INJ_STR_CA 固定 (0°)** | 噴射開始角がサイクルリセット直後に固定。高負荷時の最適噴射タイミングとの乖離可能性あり |
| **50ms タイムアウト cycleReset** | 低回転 (<600 RPM) ではサイクル周期 >100ms となりタイムアウトが誤トリガするリスク |
| **AFR補正未実装** | 空燃比フィードバックなし。燃料係数は実測逆算値のみに依存 |
| **始動時 MAP 固定値** | `start_INJ_time=50`, `start_IGN_CA=75` のみ。冷間/温間始動特性は未評価 |

#### 電気・機械環境面

| 項目 | 懸念事項 |
| ---- | -------- |
| 車載ノイズ | 点火コイル・インジェクタの誘導負荷によるGPIOノイズ未評価 |
| 電源変動 | 12V始動時の瞬低によるリセット・誤動作リスク |
| センサ信号品質 | 振動下でのエンコーダ/カムセンサ誤カウントリスク |
| ウォッチドッグ | 未実装。プログラムハング時のフェールセーフなし |

### 7-4. 総合判断

**「タイミング制御ロジック自体に問題はない」と確認できたが、「実エンジン制御に問題がない」と断言するには以下の前提条件と実走行確認が必要。**

#### 実走行移行の前提条件

- 駆動回路（MOSFET/イグナイタ）の絶縁・保護が実装済みであること
- 電源ラインのノイズ対策（フィルタ・レギュレータ）が施されていること

#### 実走行で優先確認すべき項目

1. **始動性** — スタータON時の初爆・継続燃焼の安定性
2. **低負荷巡航時の空燃比** — 排気色・燃費から空燃比を推定
3. **低回転域 (600 RPM前後) の動作安定性** — cycleResetタイムアウトの誤トリガ有無
